# Catholic Journey 365 Development Rules

## Project
React + Supabase PWA.
Production: catholicjourney365.com

## Workflow
- Never inspect the whole repo unless requested.
- Use git status first.
- Make scoped changes only.
- Prefer editing known files.
- Commit after each completed feature.
- Run npm build before commits.

## Catholic Content Rules
- Catholic accuracy first.
- Worship belongs to God alone.
- Saints intercede only.
- Mary points to Christ.
- Separate Scripture, Tradition, and private revelation.
- Do not use em dashes.

## Coding Rules
- No unnecessary refactors.
- Do not rename working components.
- Do not rewrite stable systems.
- Preserve existing UI style.
- Preserve mobile-first PWA design.

## Large Files
- Do not read full Bible JSON files.
- Do not load large datasets into context.
- Create loaders/utilities instead.
